Accessibility

Built on a native input[type="checkbox"]. The browser handles checked/unchecked state announcements automatically. The indeterminate state must be set via JavaScript — it has no HTML attribute equivalent. Every checkbox must have an accessible name via a visible <label>, aria-label, or aria-labelledby.

InteractionBehaviour
Tab / Shift+Tab Moves focus to and from the checkbox. Native <input> participates in the tab order automatically.
Space Toggles checked state when focused. Handled natively — no JavaScript needed.
aria-checked Set automatically by the browser from the checked and indeterminate properties. Indeterminate maps to aria-checked="mixed" — screen readers announce it as "mixed" or "partially checked".
indeterminate A JavaScript-only property (el.indeterminate = true). Visually shows a dash and is announced as "mixed" by screen readers. Commonly used for "select all" controls where only some children are checked.
aria-disabled Set automatically from the HTML disabled attribute. Disabled checkboxes remain in the reading order so screen reader users know the option exists.

Properties

Attribute Type Default Description
checked boolean false Sets the initial checked state.
indeterminate boolean (JS) false Shows a dash indicating partial selection. Set via el.indeterminate = true — no HTML attribute equivalent.
disabled boolean false Prevents interaction. Box renders in the disabled colour.
id string — Required when using a separate <label for="...">.
name string — Form field name submitted with the form.
value string "on" Value submitted when the checkbox is checked.

Code

<!-- Standalone checkbox -->
<span class="checkbox">
  <input type="checkbox" aria-label="Option name">
  <span class="checkbox-box">
    <svg class="checkbox-check" viewBox="0 0 10.7398 8.33382" fill="none" xmlns="http://www.w3.org/2000/svg" aria-hidden="true">
      <path d="M9.44038 0.240068...Z" fill="currentColor"/>
    </svg>
    <span class="checkbox-dash"></span>
  </span>
</span>

<!-- Checkbox with label -->
<label class="checkbox-item">
  <span class="checkbox">
    <input type="checkbox">
    <span class="checkbox-box">
      <svg class="checkbox-check" viewBox="0 0 10.7398 8.33382" fill="none" xmlns="http://www.w3.org/2000/svg" aria-hidden="true">
        <path d="M9.44038 0.240068...Z" fill="currentColor"/>
      </svg>
      <span class="checkbox-dash"></span>
    </span>
  </span>
  <span class="checkbox-item-text">Option label</span>
</label>

<!-- Indeterminate (JS required) -->
<script>
  document.querySelector('#my-checkbox').indeterminate = true;
</script>

Accessibility

Built on a native input[type="radio"]. The browser handles selection state and mutual exclusivity automatically within a name group. Every radio must have an accessible name via a visible <label>, aria-label, or aria-labelledby.

InteractionBehaviour
Tab / Shift+Tab Moves focus into and out of the radio group. Within a group, only the selected radio (or the first if none is selected) receives tab focus.
Arrow keys Moves focus and selection between options within the group. No JavaScript needed — the browser handles this natively.
Space Selects the focused radio button when it is not already selected.
name Groups radio buttons into a mutually exclusive set. Required — radios without a shared name act as independent checkboxes.
aria-disabled Set automatically from the HTML disabled attribute. Disabled radios remain in the reading order so screen reader users know the option exists.

Properties

Attribute Type Default Description
name string — Groups radios into a mutually exclusive set. All buttons in the same group share the same name.
checked boolean false Sets the initially selected option.
disabled boolean false Prevents interaction. Renders in the disabled colour but remains visible.
value string — Value submitted with the form when this option is selected.
id string — Required when using a separate <label for="...">.

Code

<span class="radio">

  <span class="radio-circle">

  </span>
</span>

Accessibility

The switch is built on a native input[type="checkbox"] with role="switch". The browser automatically exposes the checked state to assistive technology. Every toggle must have an accessible name — either via a visible <label>, aria-label, or aria-labelledby.

InteractionBehaviour
Tab / Shift+Tab Moves keyboard focus to and away from the toggle. Because this component uses a native <input>, it participates in the page tab order automatically — no extra implementation needed.
Space Flips the switch between on and off when focused. Again, native <input type="checkbox"> behaviour — the browser handles it for free.
role="switch" An ARIA attribute you add to the <input> element. It changes how screen readers (VoiceOver, NVDA, TalkBack) announce the component. Without it, a screen reader says "checkbox, checked". With it, it says "switch, on" — which is semantically correct for a toggle. One attribute, meaningful difference for blind users.
aria-checked Tells screen readers whether the switch is currently on or off. You never write this yourself — the browser sets it automatically based on the checked state of the native input and keeps it in sync on every change.
aria-disabled Tells screen readers that this control exists but cannot be interacted with. The browser sets it automatically from the HTML disabled attribute. Crucially, disabled elements are not removed from the reading order — VoiceOver will still announce them, but describe them as "dimmed" or "unavailable", so the user knows the option exists but isn't available to them.
aria-label A plain-text string that gives the toggle an accessible name when there is no visible label in the UI. Screen readers read this string aloud when the user navigates to the toggle. Example: aria-label="Enable dark mode". If the toggle already has a visible <label> element associated via for, this attribute is not needed.

Properties

Attribute Type Default Description
checked boolean — Sets the initial checked (on) state. Omit for off.
disabled boolean false Prevents interaction. Track and icon render in the disabled colour.
id string — Required when the label is a separate element using for / aria-labelledby.
name string — Form field name submitted with the form.
value string "on" Form field value submitted when the switch is on.

Code

<!-- Basic switch -->
<label class="toggle-switch">
  <input type="checkbox" role="switch" aria-label="Setting name">
  <span class="toggle-track">
    <span class="toggle-handle">
      <span class="toggle-icon">
        <svg class="icon-check" width="12" height="12" viewBox="120 56 48 48" fill="none" xmlns="http://www.w3.org/2000/svg" aria-hidden="true">
          <path fill-rule="evenodd" clip-rule="evenodd" d="M160.305 65.1024...Z" fill="currentColor"/>
        </svg>
        <svg class="icon-x" width="12" height="12" viewBox="56 56 48 48" fill="none" xmlns="http://www.w3.org/2000/svg" aria-hidden="true">
          <path fill-rule="evenodd" clip-rule="evenodd" d="M65.7741 98.467...Z" fill="currentColor"/>
        </svg>
      </span>
    </span>
  </span>
</label>

<!-- With visible label (settings row pattern) -->
<div style="display:flex; align-items:center; justify-content:space-between; gap:24px;">
  <div>
    <label for="ts-setting" style="font-weight:600;">Push notifications</label>
    <p style="font-size:14px; color:var(--color-content-subtle);">Receive alerts when something requires your attention.</p>
  </div>

    <span class="toggle-track">...</span>

  <span class="toggle-track">...</span>

Accessibility

Built on the ARIA tablist / tab pattern. The <nav> element provides a landmark so screen reader users can jump directly to the navigation. Only the active tab is in the tab order at any time (roving tabindex) — arrow keys move between tabs without leaving the component.

InteractionBehaviour
Tab / Shift+Tab Moves focus into and out of the tab bar. Only the currently active tab receives tab focus — inactive tabs have tabindex="-1" and are skipped.
← → Arrow keys Cycle focus and selection left or right through the tabs with wrapping. This is the standard tablist navigation pattern — users do not need to Tab through every option.
role="tablist" Applied to .bnav-track. Groups the tab buttons into a single navigable set and tells screen readers this is a tab bar, not a list of unrelated buttons.
role="tab" Applied to each .bnav-tab button. Screen readers announce the tab's accessible name (from aria-label) together with its selected state and position — e.g. "Home, tab, selected, 1 of 4".
aria-selected Set to "true" on the active tab and "false" on all others. Updated by initBnav on every selection change so the state is always in sync with the visual indicator.
aria-label on <nav> Names the navigation landmark. Without it VoiceOver announces "navigation" generically; with it, it announces the specific name (e.g. "Main navigation"). Required when a page has more than one <nav>.
aria-label on FAB The FAB has no visible text label, so aria-label is required to describe its action. Describe what it does, not what it looks like — e.g. aria-label="New post" not "Plus button".

Properties

ClassValuesDescription
bnav—Root <nav> element. Flex row containing the track and optional FAB.
bnav-track—Pill-shaped tab container on a div. Use position:relative — the animated pill is absolutely positioned inside.
bnav-pill—Sliding selection indicator on a span. Position and width are driven by initBnav on every tab change.
bnav-tab—Individual tab button. Always 40 × 40 px when unselected; expands to 96 px wide when bnav-tab--active is present.
bnav-tab--active—Modifier on bnav-tab. Expands the tab, fades in the label, and turns the icon to the inverse colour.
bnav-label—Tab label text in a span. Invisible (opacity 0) when unselected; fades in with a 60 ms delay when the tab becomes active.
bnav-fab—Floating action button — 56 × 56 px circle. Sits alongside the track. Requires an aria-label.
bnav--icon-only—Modifier on bnav. Hides all labels; the selected tab stays 40 × 40 px and the pill slides without expanding.
bnav--reversed—Modifier on bnav. For use on dark surfaces — inverts track and pill colours.

Code

<nav class="bnav" aria-label="Main navigation">
  <div class="bnav-track">
    <span class="bnav-pill" aria-hidden="true"></span>

    <button class="bnav-tab bnav-tab--active" data-bnav-tab
            aria-label="Home" aria-selected="true">
      <span class="bnav-icon" aria-hidden="true"><!-- SVG --></span>
      <span class="bnav-label">Home</span>
    </button>

    <button class="bnav-tab" data-bnav-tab
            aria-label="Search" aria-selected="false">
      <span class="bnav-icon" aria-hidden="true"><!-- SVG --></span>
      <span class="bnav-label">Search</span>
    </button>

    <!-- repeat for each tab -->
  </div>

  <button class="bnav-fab" aria-label="New post">
    <!-- SVG -->
  </button>
</nav>

Accessibility

Badges are purely visual — they carry no semantic meaning on their own. The surrounding interactive element (button, link, icon) must communicate the badge's meaning to screen readers via aria-label. Never rely on the badge alone to convey critical state.

PatternGuidance
Badge on a button Include the count in the button's aria-label: <button aria-label="Notifications, 5 unread">. The badge span should have aria-hidden="true".
Dot badge Always pair with a meaningful label on the parent: e.g. aria-label="Messages (new)". A dot alone gives no information.
Live updates If the count updates dynamically, wrap the badge in a role="status" or aria-live="polite" region so changes are announced.
Colour contrast Both info and alert backgrounds meet WCAG AA against white text at all supported sizes.

Properties

ClassValuesDescription
badge-dot — Base dot badge. A solid circle with a white separation ring.
badge-dot--{size} small · large 8px (small) or 12px (large) diameter.
badge-dot--alert — Switches fill to --color-danger. Default (info) uses --color-info.
badge-number — Base number badge. Pill-shaped container with semibold text.
badge-number--{size} small · large 18px height (small) or 24px height (large). Expands horizontally for multi-digit counts.
badge-number--alert — Switches fill to --color-danger. Default (info) uses --color-info.
badge-host — Wrapper for positioning a badge over another element. Set badge-host-badge on the badge itself.

Code

<!-- Dot badge -->

<!-- Number badge -->

<span class="badge-number badge-number--large badge-number--alert">99+</span>

<!-- Positioned over another element -->

  <div class="avatar avatar--large avatar--circle">

  <svg>...bell icon...</svg>

Accessibility

Avatars are purely presentational. The <img> element should have a descriptive alt attribute when the user's identity is meaningful in context (e.g. "Sofia Esposito's avatar"), or an empty alt="" when the avatar is decorative and the user's name appears nearby. The initials variant is wrapped in a span with aria-hidden="true" — the surrounding element should carry the accessible name.

CaseGuidance
Photo with nearby name Use alt="" — the name in the UI provides the accessible label.
Photo without nearby name Use alt="[Name]'s avatar" so screen readers can identify the person.
Initials fallback Hide the initials with aria-hidden="true" and provide the full name via aria-label on the parent container.
Clickable avatar Wrap in an <a> or <button> and provide a label that describes the action, e.g. "View Sofia's profile".

Properties

Class Values Description
avatar--{size} xsmall · small · medium · large · xlarge · xxlarge Sets the avatar dimensions: 24 / 32 / 40 / 48 / 56 / 64 px.
avatar--{shape} circle · squircle · diamond · cushion Applies the clip-path for each shape variant.
avatar-img — Photo variant. Uses object-fit: cover — works with any aspect ratio.
avatar-initials — Initials fallback. Renders semibold text on a café-200 background, sized relative to the avatar.

Code
